使用Hexo最快部署个人博客，本地测试运行

发表于 2025-11-09 更新于 2026-01-26 阅读次数：

本文会讨论如何使用Hexo搭建并部署个人博客，并在本地测试运行。

大多数人在想第一次创建属于自己的个人博客时都会为了追求完美而选择各种博客框架。其中，功能强大的往往令人难以入手，而过于简单的又怕无法满足所需。

我的建议是：无论你的背景如何，都应从 Hexo 开始创建你的第一篇个人博客

选择Hexo的原因是，它拥有最平滑的起步曲线，足以让你在几分钟内见证第一个博客的诞生。同时，它也包含成熟的技术框架和海量的主题框架。而在你学习完Hexo最基础的那一部分后，你已有能力自己对博客进行拓展和自定义，无需受限于别人提供的配置。

Hexo 是一个基于 Node.js 的静态博客框架。它的工作原理很简单：你将用 Markdown 写的文章，通过 Hexo 生成为一整套快速的静态网页。这意味着你无需担心数据库和服务器安全，可以免费托管在 GitHub Pages、Netlify 等平台，甚至可以部署到你自己的云服务器上，并且拥有极快的访问速度。

接下来，我们将逐渐带你完成从零到一的搭建过程。

安装Hexo并创建博客

在以下的步骤中，都会以Linux环境为例运行命令，创建文件夹等命令应根据你的系统更改。

前置条件：

安装Hexo有一些微小的前置条件

  • NPM （Node.js 自带，安装Node.js即可获得，下载地址：nodejs.org），版本不低于10.13

  • 操作系统：Windows, macOS, Linux

  • 文本编辑器：VsCode, Notepad++，Vim，等等都可以。不建议系统自带的记事本

后面可能会用到的（不用着急安装）：

  • Git（此处 下载)

  • Github账户（官网）

  • /_config.yml Hexo配置文件，设置站点的基础信息及框架

选择方案

在 Next 中，你可以设置你喜欢的方案 Scheme（不同方案只影响布局）：

  • Muse 默认方案，简洁的单列视图

  • Mist 在 Muse 基础上更加紧凑

  • Pisces 双列方案

  • Gemini 在 Pisces 基础上添加了板块和阴影

在Next配置文件中搜索 scheme 关键词，通过移除并添加 # 注释来选择你想要的主题
